# Умный холодильник (ООП + SOLID)

## Функционал

- Авторизация пользователей (владелец / гость)
- Открыть/закрыть дверь
- Добавить продукт в зону (только владелец)
- Удалить продукт из зоны (только владелец)
- Проверить просроченные продукты
- Показать состояние зон

## Зоны

- основная
- морозилка

## Команды терминала
- login <имя>
- open <основная/морозилка>
- close <основная/морозилка>
- add <название> <цена> <срок> <год> <мес> <день> <зона>
- remove <название> <зона>
- get_expired <год> <мес> <день> <зона>
- status
- help
- exit

## Соответствие принципам SOLID
SRP
    User управляет правами, Door — состоянием двери, Refrigerator — координацией зон и авторизацией, main.py — интерфейсом.
OCP
    Новые зоны добавляются расширением логики add/remove без изменения существующих классов.
LSP
    Все пользователи (User) взаимозаменяемы через единый интерфейс с атрибутом is_owner.
ISP
    Операции разделены: управление дверью (open_door/close_door), изменение содержимого (add/remove), чтение состояния (__str__, get_expired).
DIP
    Refrigerator зависит от абстракций (User, Door), а не от конкретных реализаций. Внешний код зависит от интерфейса Refrigerator.
